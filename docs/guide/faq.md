### 为什么组件不全？没有表格、面包屑、轮播？
我们的组件库主要的目标是 C 端应用。大家根据 UI 的设计也能看出，并不适合开发管理后台等应用场景。所以我们最先设计了大部分 C 端会用到的组件，而表格、面包屑这种大部分在管理后台上使用的组件没有优先设计和开发。

当然，这不代表我们以后不会做。

至于轮播，我想我们做一个功能相对简单的轮播也不如你使用 swiper 来的容易。而且 swiper 也很好用，这一部分请参考[最佳实践](/usage/best-pracitces)。
